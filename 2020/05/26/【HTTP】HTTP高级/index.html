Cookie 和 Session 的区别

  1. 存储位置上:Cookie以文本文件格式存储在浏览器中，而session存储在服务端它存储了限制数据量。它只允许4kb它没有在cookie中保存多个变量。
  2. 存储量上:cookie的存储限制了数据量，只允许4KB，而session是无限量的
  3. 安全上: 我们可以轻松访问cookie值但是我们无法轻松访问会话值，因此它更安全
  4. 生命周期上:设置cookie时间可以使cookie过期。但是使用session-destory（），我们将会销毁会话。

Cookie 和 LocalStorage的区别

1.生命周期上:Cookie可以自己设置默认到浏览器关闭，而LocalStorage 除非自行删除或清除缓存，否则一直存在。
2.存储量上:cookie的存储限制了数据量，只允许4KB，而LocalStorage是5MB
3.与服务器通信上:Cookie在http头中，而LocalStorage在客户端上
4.易用性上:Cookie蹩脚的接口，一般要自己去封装，而LocalStorage接口可以直接使用。

SessionStorage 和 LocalStorage 的区别

1.生命周期上:SessionStorage在浏览器关闭，而LocalStorage 除非自行删除或清除缓存，否则一直存在。
2.与服务器通信上:SessionStorage不参与服务器通信，而LocalStorage在客户端上

Cache-Control: max-age=1000 缓存 与 ETag 的「缓存」有什么区别？

Cache-Control: max-age=1000 缓存，是1000秒内不返回请求
ETag 的「缓存」，是返回请求响应体为空

两种最主要的区别就是Cache-Control缓存存储的有效周期内不会发请求相同URL的资源不会发请求，但是使用ETag浏览器还是会发请求。
