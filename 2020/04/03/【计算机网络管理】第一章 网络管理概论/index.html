二、网络管理的基本概念

2.1 计算机网络管理的定义

采用计算机硬软件技术对客户端计算机、服务器、存储和交换机、路由器等网络设备及相关软件组成的网络和信息系统进行管理的工作。

2.2 网络管理的任务

  • 监视网络状态
  • 控制网络运行

2.3 网络管理需要解决的问题

  • 复杂性
  • 可靠性
  • 安全性
  • 可扩充性

2.4 网络管理的根本目标

  • 有效性–准确及时传递信息，提供服务
  • 可靠性–持续稳定运行
  • 开放性–兼容各个厂商不同类型设备
  • 综合性–提供各种综合业务功能
  • 安全性–对网络中的信息进行保护
  • 经济性–减少运行费用

2.5 网络管理的对象

网络管理涉及的资源–被管对象

  • 硬件资源
    • 物理介质
    • 计算机设备
    • 网络设备
  • 软件资源
    • 操作系统
    • 应用软件
    • 通信软件

被管对象的集合称为管理信息库（MIB）

问题：网络管理中被管理的硬件资源有哪些？

  1. 物理介质：包括物理层和数据链路层联网设备，如网卡、双绞线、光纤等。
  2. 计算机设备：包括处理机、打印机和存储设备以及其他计算机外围设备。
  3. 网络互联设备：如网桥、路由器、网关等。

3.2 网络运行管理模式

  • 集中式网络管理模式
  • 分布式网络管理模式
  • 混合管理模式

问题 对比集中式网络管理和分布式网络管理的区别以及各自的优缺点

  • 集中式
    • 优点是结构简单，易于维护，管理集中，可以对整个网络系统全局进行有效管理。
    • 确定是信息流集中于管理节点，易产生瓶颈，管理不够灵活，可扩展性差。
  • 分布式
    • 优点是扩展性好，稳固可靠，能提高网络性能。
    • 确定是管理和维护比较复杂。

4.2 配置管理

配置管理的主要功能

  • 发行软件
  • 检查参数值、互联关系和报告配置现状
