简单理解HTTP之请求与响应

发表于2020-04-03更新于2020-04-03阅读次数

定义

超文本传输协议（英语：HyperText Transfer Protocol，缩写：HTTP）是一种用于分布式、协作式和超媒体信息系统的应用层协议。HTTP是万维网的数据通信的基础。

HTTP 的作用就是指导浏览器和服务器如何进行沟通。

curl命令

在Linux中curl是一个利用URL规则在命令行下工作的文件传输工具，可以说是一款很强大的http命令行工具。它支持文件的上传和下载，是综合传输工具，但按传统，习惯称url为下载工具。

语法

curl [option] [url]

option

  • -H/–header <line> 自定义头信息传递给服务器
  • -s/–silent 静音模式。不输出任何东西
  • -V/–version 显示版本信息
  • -X/–request <command> 指定什么命令
  • d/–data <data> HTTP POST方式传送数据

HTTP请求

组成部分

请求最多包含四部分，最少包含三部分。（也就是说第四部分可以为空）
第三部分永远都是一个回车
动词有 GET POST PUT PATCH DELETE HEAD OPTIONS 等
这里的路径包括「查询参数」，但不包括「锚点」
如果没有写路径，那么路径默认为 /
第2部分中的 Content-Type 标注了第 4 部分的格式

例子

命令curl -s -v -H “qqq111: xxx” – “https://www.baidu.com"

如何用Chrome开发者工具查看 HTTP 请求内容

1.点右键
2.点“检查”
3.点“Network”那栏
4.在输入框输入baidu.com
5.在Network那里会获取相关http信息，找到Name下的www.baidu.com选项点击进去
6.找到Request Headers选项
7.点击隔壁的view source
8.就可以查看到HTTP请求内容

HTTP响应

组成部分

状态码

1xx 不常用
2xx 表示成功 200
3xx 表示滚吧，301 302
4xx 表示客户端错误 404
5xx 表示好吧，服务器错误 502

例子

命令 curl -X POST -d "1234567890" -s -v -H "qqq111: xxx" -- "https://www.baidu.com"

HTTP 响应包括哪些部分，如何用Chrome开发者工具查看 HTTP 响应内容

1.点右键
2.点“检查”
3.点“Network”那栏
4.在输入框输入baidu.com
5.在Network那里会获取相关http信息，找到Name下的www.baidu.com选项点击进去
6.找到Response Headers选项
7.点击隔壁的view source
8.就可以查看到HTTP响应内容
